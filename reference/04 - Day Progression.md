# Day and Difficulty Progression — Approved v1

## Day 0 — Training

Teach the core action set with obvious cases:

- Approve;
- Remove;
- Escalate;
- harmless posts;
- obvious spam/scams;
- simple insults;
- clear threats.

The player is not required to select a detailed escalation reason.

## Days 1–2 — Core moderation

Use straightforward examples involving:

- Normal and Lifestyle posts;
- Spam and Fraud;
- Harassment;
- Privacy;
- Threats;
- Misinformation;
- Public Information and News / Public Event posts.

The correct primary reason should still exist in JSON even when the UI does not yet ask the player to select it.

## Day 3 — Parent escalation reasons unlock

Show the original first-level menu:

- Threat / Violence;
- Self-Harm / Crisis;
- Harassment / Abuse;
- Fraud / Scam;
- Privacy / Sensitive Info;
- Misinformation / Public Harm;
- Needs Review where permitted.

No specialist subreason tree is required yet.

## Days 4–5 — Borderline and contextual harm

Introduce:

- unpleasant but allowed opinions;
- satire and hyperbole;
- personal self-harm crisis versus harmful promotion;
- direct hate content versus discussion;
- adult explicit content versus non-consensual abuse;
- repeated harassment or stalking;
- private-message context;
- mal-information;
- graphic violence handled through Threat / Violence.

## Day 6 — Account metadata and patterns

Use:

- account age;
- follower/following ratios;
- previous flags;
- verification;
- post count and bio;
- repeated usernames;
- related accounts;
- coordinated posting patterns.

Metadata should support a judgement, not function as a secret answer key.

## Day 7 onward / specialist mode

The same six parent escalation buttons remain. Selected parents may open a second-level specialist menu.

Planned examples:

- Misinformation / Public Harm → synthetic media, public health, civic manipulation, coordinated disinformation;
- Threat / Violence → credible threat, incitement, graphic violence, extremist violence;
- Harassment / Abuse → stalking, sustained abuse, brigading, hate abuse;
- Privacy / Sensitive Info → location, medical, identity, financial or intimate information;
- Fraud / Scam → investment, credential, impersonation or marketplace fraud.

Exact specialist unlock days may be adjusted during balancing, but filler generation must not introduce a specialist mechanic before the player has the evidence and UI needed to reason about it.

## Synthetic-media progression

- Early: manipulated media is categorised and answered under Misinformation / Public Harm.
- Later: Synthetic Media may become a separate post category and a branch under Misinformation / Public Harm.
- Self-contained cases are required until recurring characters and public figures exist in the story.
- The player must receive provenance, account or narrative evidence rather than being asked to spot visual artefacts.

## Hate/extremism progression

- Early: route by dominant existing parent harm.
- Later: distinguish targeted hate abuse, extremist violence/recruitment and coordinated propaganda as specialist branches.

## Spam/platform-manipulation progression

- Early ordinary spam: Remove.
- Later coordinated networks: use metadata and route by purpose, with a future Platform Manipulation branch if implemented.
